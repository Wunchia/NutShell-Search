# NutShell Search — 搜索引擎项目技术文档

## 项目概述

NutShell Search 是一个基于 C++17 实现的全文搜索引擎，采用离线建库 + 在线检索的经典架构。项目分三期开发，涵盖中文分词、TF-IDF 权重计算、倒排索引、SimHash 去重、Trie 前缀索引、LRU 缓存、Redis 二级缓存、muduo 网络框架、TLV/HTTP 双协议服务等核心技术。

### 技术栈

| 层次 | 技术 | 用途 |
|------|------|------|
| 语言 | C++17 | 核心开发语言 |
| 构建 | CMake 3.15+ | 跨平台构建 |
| 分词 | cppjieba (MIX 模式) | 中文分词 |
| 去重 | simhash (Jenkins Hash) | 文档去重 (汉明距离 ≤ 3) |
| 字符处理 | utfcpp | UTF-8 字符遍历与码点提取 |
| XML 解析 | tinyxml2 | 语料解析 + pages.dat 按需读取 |
| 网络框架 | muduo (Reactor 模型) | TCP/HTTP 服务器 |
| JSON | nlohmann_json | 数据序列化 |
| 缓存 | hiredis + 自研 LRU | L1 进程内缓存 + L2 Redis |
| 前端 | 原生 HTML/CSS/JS | 搜索界面 (毛玻璃效果) |
| 日志 | muduo Logger | 分级日志 (DEBUG/INFO/WARN/ERROR) |

### 1.2 中文关键字推荐离线流程

#### 数据流

```
corpus/CN/*.txt (语料文件)
    │
    ▼ DirectoryScanner::scan()          遍历目录，获取文件列表
    │
    ▼ istreambuf_iterator + cppjieba    全文读入 → MIX 模式分词
    │
    ▼ 去停用词 + is_chinese_punctuation 过滤无意义 token
    │
    ▼ std::map<string,int>              统计词频
    │
    ▼ 输出 dict_cn.dat                  "词 词频"
    │
    ▼ TextUtils::split_utf8_characters  将词拆成单字
    │
    ▼ std::map<string, set<int>>        单字 → 行号集合（倒排索引）
    │
    ▼ 输出 index_cn.dat                 "字 行号1 行号2..."
```

#### 核心代码

```cpp
// src/offline/KeywordProcessor.cpp
void KeywordProcessor::create_cn_dict(const std::string& dir,
                                      const std::string& outfile)
{
    auto files = DirectoryScanner::scan(dir);   // ① 扫描目录

    std::map<std::string, int> freq;
    for (const auto& filepath : files) {
        std::ifstream ifs(filepath);
        // ② 全文读入 (istreambuf_iterator: 一次读完整文件, 不按行处理)
        std::string content((std::istreambuf_iterator<char>(ifs)),
                             std::istreambuf_iterator<char>());

        std::vector<std::string> words;
        JiebaSingleton::getJieba().Cut(content, words);  // ③ MIX 分词

        for (const auto& word : words) {
            if (_cnStopWords.count(word) > 0) continue;        // ④ 去停用词
            if (TextUtils::is_chinese_punctuation_or_space(word))
                continue;                                       // ⑤ 去标点
            freq[word]++;                                       // ⑥ 词频++
        }
    }

    // ⑦ 写入 dict_cn.dat
    std::ofstream ofs(outfile);             // 格式: 词 词频\n
    for (const auto& [word, count] : freq)
        ofs << word << " " << count << "\n";
}
```

```cpp
void KeywordProcessor::build_cn_index(const std::string& dict,
                                      const std::string& index)
{
    std::map<std::string, std::set<int>> charIndex;

    std::ifstream ifs(dict);
    std::string word;
    int freq;
    int lineNo = 1;

    while (ifs >> word >> freq) {
        // ⑧ utfcpp 拆单字: "花生" → ["花", "生"]
        auto chars = TextUtils::split_utf8_characters(word);
        for (const auto& ch : chars)
            charIndex[ch].insert(lineNo);   // ⑨ 单字倒排: 字→行号
        lineNo++;
    }

    std::ofstream ofs(index);               // ⑩ 写入 index_cn.dat
    for (const auto& [ch, lines] : charIndex) {
        ofs << ch;                          // 格式: 汉字 行号1 行号2...
        for (int line : lines) ofs << " " << line;
        ofs << "\n";
    }
}
```

```cpp

{
```

### 1.3 英文关键字推荐离线流程

#### 数据流

```
corpus/EN/*.txt
    │
    ▼ DirectoryScanner::scan()
    │
    ▼ normalize_english_line()             去标点数字 → 小写 → 替换为空格
    │                                      例: "Hello, World!" → "hello  world"
    │
    ▼ istringstream >> word                空白字符自动分词
    │
    ▼ 去停用词
    │
    ▼ std::map<string,int>                 统计词频
    │
    ▼ 输出 dict_en.dat                     "词 词频"
    │
    ▼ 全量前缀索引 → 去冗                   例: apple → a/ap/app/appl/apple
    │                                      去冗: 只保留行号集合不同的前缀
    │
    ▼ Trie 序列化 → index_en.dat (二进制)   紧凑存储，树形结构
```

#### 前缀去冗算法

```
假设词库只有 apple、apply、banana，行号分别为 1、2、3:

全量前缀:
  a→{1,2}  ap→{1,2}  app→{1,2}  appl→{1,2}  apple→{1}
  appl→{1,2}  apply→{2}  b→{3}  ba→{3}  ban→{3}  bana→{3}  banan→{3}  banana→{3}

去冗后 (行号集合不变的前缀被淘汰):
  a→{1,2}  apple→{1}  apply→{2}  b→{3}
  ↑ 因为 ap/app/appl 的行号都是 {1,2}, 和 a 一样 → 冗余, 删掉
  ↑ 因为 ba/ban/bana/banan/banana 的行号都是 {3}, 和 b 一样 → 冗余, 删掉
```

```cpp
// src/offline/KeywordProcessor.cpp: build_en_index()
// 第一遍: 构建全量前缀索引
std::map<std::string, std::set<int>> rawIndex;
while (ifs >> word >> freq) {
    for (size_t len = 1; len <= word.size(); ++len)
        rawIndex[word.substr(0, len)].insert(lineNo);
    lineNo++;
}

// 第二遍: 去冗 — 行号集合不变的前缀丢弃
std::map<std::string, std::set<int>> prefixIndex;
std::string lastKept;
for (auto& [pref, lines] : rawIndex) {
    if (pref.size() == 1) {
        prefixIndex[pref] = std::move(lines);   // 单字前缀必保留
        lastKept = pref;
        continue;
    }
    if (prefixIndex[lastKept] != lines) {       // 行号集合不同 → 分叉
        prefixIndex[pref] = std::move(lines);   // 保留
        lastKept = pref;
    }
    // 相同 → 跳过 (冗余)
}
```

#### Trie 序列化

将去冗后的前缀集构建成 Trie，以紧凑二进制格式写入 `index_en.dat`：

```cpp
// include/common/Trie.h — 序列化格式:
// [int32:行号数量][int32×N:行号][int32:子节点数量][char:字符]+递归...

void Trie::serialize(std::ostream& os) const {
    serializeNode(&_root, os);
}

static void serializeNode(const Node* node, std::ostream& os) {
    int32_t lc = node->lineNos.size();
    os.write(&lc, 4);                     // 行号个数
    for (int lineNo : node->lineNos) {
        int32_t ln = lineNo;
        os.write(&ln, 4);                 // 每个行号 (4 字节)
    }
    int32_t cc = node->children.size();
    os.write(&cc, 4);                     // 子节点个数
    for (const auto& [c, child] : node->children) {
        os.put(c);                        // 字符 (1 字节)
        serializeNode(child.get(), os);   // 递归写子树
    }
}
```

> 序列化文件体积约为原文本格式的 40%，加载时直接反序列化重建树结构。

### 1.4 网页搜索离线流程

#### 数据流

```
corpus/webpages/*.xml (RSS 格式)
    │
    ▼ tinyxml2 解析 <rss>→<channel>→<item>
    │  提取 link / title / content (fallback: description)
    │  → vector<Document>
    │
    ▼ SimHash 去重                            topN=max(5, min(200, size/120))
    │  计算每篇文档的 64-bit SimHash 指纹
    │  与已有指纹逐条比较汉明距离 ≤ 3 → 重复, 丢弃
    │
    ▼ build_pages_and_offsets()
    │  序列化 XML: <doc><id>..</id><link>..</link>...
    │  CDATA 包裹 content: <content><![CDATA[...]]></content>
    │  → pages.dat (网页库) + offsets.dat (偏移库)
    │
    ▼ cppjieba 分词 + TF-IDF
    │  第一遍: 统计 TF (词频) + DF (文档频率)
    │  第二遍: 计算 ID=log2(N/(DF+1)); weight=TF×IDF
    │  归一化: weight / √(Σ weight²)
    │
    ▼ 输出 invert_index.dat                   "关键词 docId weight ..."
```

#### SimHash 去重

```cpp
// src/offline/PageProcessor.cpp
void PageProcessor::deduplicate_documents()
{
    if (_documents.empty()) return;

    std::vector<Document> uniqueDocs;
    std::vector<uint64_t> fingerprints;

    for (auto& doc : _documents) {
        uint64_t hash = 0;
        // topN 动态调整: 文本越长, 取的关键词越多
        int topN = std::max(5, std::min(200,
            static_cast<int>(doc.content.size() / 120)));
        _hasher.make(doc.content, topN, hash);  // simhash 内部: 分词→加权→降维

        bool duplicated = false;
        for (size_t i = 0; i < fingerprints.size(); ++i) {
            if (simhash::Simhasher::isEqual(hash, fingerprints[i])) {
                duplicated = true;                // 汉明距离 ≤ 3 → 重复
                break;
            }
        }

        if (!duplicated) {
            uniqueDocs.push_back(std::move(doc));
            fingerprints.push_back(hash);
        }
    }
    _documents = std::move(uniqueDocs);
}
```

> **SimHash 原理**: 对文档进行分词 → 给每个词计算传统 hash (Jenkins) → 加权累加 → 二值化 →
> 得到 64-bit 指纹。两篇相似文档的指纹汉明距离很小 (≤ 3)，不相似的距离大。

#### TF-IDF 权重计算

```cpp
// src/offline/PageProcessor.cpp: build_inverted_index()
// 第一遍: 统计 TF + DF
std::map<int, std::map<std::string, int>> docTermCount;   // 文档id→(词→次数)
std::map<std::string, int> documentFrequency;              // 词→包含该词的文档数

for (const auto& doc : _documents) {
    std::vector<std::string> words;
    JiebaSingleton::getJieba().Cut(doc.content, words);

    std::map<std::string, int> localFreq;
    for (const auto& w : words) {
        if (_stopWords.count(w) > 0) continue;
        if (TextUtils::is_chinese_punctuation_or_space(w)) continue;
        localFreq[w]++;           // 词频 (TF)
    }
    docTermCount[doc.id] = localFreq;
    for (const auto& [word, _] : localFreq)
        documentFrequency[word]++;  // 文档频率 (DF) — 每文档只计一次
}

// 第二遍: 计算 TF-IDF + 归一化
for (const auto& doc : _documents) {
    int totalWords = docTotalWords[doc.id];
    const auto& localFreq = docTermCount[doc.id];

    std::map<std::string, double> rawWeights;
    double squareSum = 0.0;

    for (const auto& [word, cnt] : localFreq) {
        // TF = 词在本文档中的占比
        double tf = static_cast<double>(cnt) / totalWords;
        // IDF = 逆文档频率 (出现该词的文档越少, IDF 越大)
        double idf = std::log2(static_cast<double>(N) /
                               (documentFrequency[word] + 1));
        double weight = tf * idf;            // 原始权重
        rawWeights[word] = weight;
        squareSum += weight * weight;        // 平方和 (归一化用)
    }

    double norm = std::sqrt(squareSum);      // 归一化分母
    for (const auto& [word, weight] : rawWeights) {
        double normalized = (norm > 0) ? weight / norm : 0.0;
        _invertedIndex[word][doc.id] = normalized;  // 归一化后的权重
    }
}

// 写入 invert_index.dat: "关键词 docId1 weight1 docId2 weight2 ..."
```

> **归一化的目的**: 让每篇文档的向量长度变为 1 (|Y|=1)，这样余弦相似度 cosθ = X·Y，
> 查询时只需点积，无需再算模长。

### 2.2 TLV 协议

解决 TCP 粘包问题的自定义协议。每条消息 = 5 字节头 + 变长体:

```
┌──────────┬──────────────┬────────────────┐
│ type (1B)│ length (4B)  │ value (length B)│
│ 1=keywd  │ (网络序)     │  实际数据       │
│ 2=search │              │                │
└──────────┴──────────────┴────────────────┘
```

```cpp
// include/online/TlvCodec.h
struct Message {
    uint8_t  type;    // 消息类型: 1=关键字推荐, 2=网页搜索
    std::string value;// 消息内容 (JSON)
};

// src/online/TlvCodec.cpp — 解码逻辑
void TlvCodec::onMessage(const TcpConnectionPtr& conn,
                         Buffer* buf, Timestamp time)
{
    while (buf->readableBytes() >= kHeaderLen) {    // 至少 5 字节
        // 读 header
        uint8_t  type   = *(buf->peek());
        uint32_t length = networkToHost32(*(uint32_t*)(buf->peek() + 1));

        if (buf->readableBytes() < kHeaderLen + length)
            break;                         // 数据没收完 → 等下次 TCP 包

        buf->retrieve(kHeaderLen);         // 消费 header

        Message msg;
        msg.type = type;
        msg.value.assign(buf->peek(), length);  // 读 value
        buf->retrieve(length);                  // 消费 value

        _callback(conn, msg, time);        // 回调业务层
    }
}
```

### 2.3 关键字推荐查询

#### 中文查询流程

```
用户输入 "花生"
    │
    ▼ containsCJK("花生") → true → queryChinese()
    │
    ▼ TextUtils::split_utf8_characters("花生") → ["花", "生"]
    │
    ▼ _idxCn["花"] ∪ _idxCn["生"]  → 候选词行号并集
    │   (索引是单字倒排: 字→行号集合)
    │
    ▼ 逐行号读 _dictCn[lineNo-1]  → (候选词, 词频)
    │
    ▼ editDistance("花生", 每个候选词) → 编辑距离
    │
    ▼ 三级排序: (dist ASC, freq DESC, word ASC) → TopK
    │
    ▼ nlohmann::json::dump() → '["花生","花生米","花生酱",...]'
```

#### 英文查询 + Trie 搜索

```cpp
// src/online/KeywordService.cpp — queryEnglish()
std::string KeywordService::queryEnglish(const std::string& keyword, int topK)
{
    // Trie::search() 已内置逐级回退：
    //   输入 "appl" → 先找完整路径 "a→p→p→l"
    //   若 "appl" 节点无数据 → 回退到最接近有效前缀 (如 "app")
    std::set<int> candidateLines = _idxEn.search(keyword);

    if (candidateLines.empty()) return "[]";

    // 用行号从词典读候选词
    struct Candidate { int dist; int freq; std::string word; };
    std::vector<Candidate> candidates;

    for (int lineNo : candidateLines) {
        const auto& [word, freq] = _dictEn[lineNo - 1];
        int dist = editDistance(keyword, word);
        candidates.push_back({dist, freq, word});
    }

    // 三级排序 + TopK → JSON
    std::sort(candidates.begin(), candidates.end(),
        [](const Candidate& a, const Candidate& b) {
            if (a.dist != b.dist) return a.dist < b.dist;  // ① 编辑距离升序
            if (a.freq != b.freq) return a.freq > b.freq;  // ② 词频降序
            return a.word < b.word;                          // ③ 字典序
        });

    nlohmann::json result = nlohmann::json::array();
    for (size_t i = 0; i < std::min(topK, candidates.size()); ++i)
        result.push_back(candidates[i].word);
    return result.dump();
}
```

#### Trie 搜索原理

```cpp
// include/common/Trie.h — search() 核心
std::set<int> search(const std::string& prefix) const {
    const Node* node = &_root;
    const Node* lastValid = &_root;      // 回退锚点

    for (char c : prefix) {
        auto it = node->children.find(c);
        if (it == node->children.end())
            return lastValid->lineNos;   // 走不下去了 → 回退

        node = it->second.get();
        if (!node->lineNos.empty())
            lastValid = node;            // 该前缀有数据 → 更新锚点
    }
    return node->lineNos.empty()
        ? lastValid->lineNos : node->lineNos;
}
```

> **时间复杂度**: O(len(prefix))，与词库大小无关。对比 `map<string,set<int>>` 的
> `find()` O(log N) × 逐级回退 O(len)，Trie 更快且代码更简洁。

#### 编辑距离（Levenshtein Distance）

```cpp
// src/online/KeywordService.cpp — 滚动数组优化, O(m×n) 时间, O(n) 空间
int KeywordService::editDistance(const std::string& s1, const std::string& s2)
{
    int m = s1.length(), n = s2.length();
    std::vector<int> prev(n + 1), curr(n + 1);

    for (int j = 0; j <= n; ++j) prev[j] = j;  // 初始化第一行

    for (int i = 1; i <= m; ++i) {
        curr[0] = i;
        for (int j = 1; j <= n; ++j) {
            int del = prev[j] + 1;              // 删除 s1[i-1]
            int ins = curr[j - 1] + 1;          // 插入 s2[j-1]
            int sub = prev[j - 1];              // 替换 / 匹配
            if (s1[i - 1] != s2[j - 1]) sub++;
            curr[j] = std::min({del, ins, sub});
        }
        std::swap(prev, curr);                  // 滚动
    }
    return prev[n];
}
```

#### 中/英文自动判断

```cpp
// src/online/KeywordService.cpp — containsCJK()
// 利用 utfcpp::iterator 遍历 Unicode 码点，检测 CJK 汉字区间
bool KeywordService::containsCJK(const std::string& s)
{
    auto it  = utf8::iterator<...>(s.begin(), s.begin(), s.end());
    auto end = utf8::iterator<...>(s.end(),   s.begin(), s.end());

    for (; it != end; ++it) {
        char32_t cp = *it;
        // CJK 统一汉字基本区 (U+4E00 ~ U+9FFF)
        if (cp >= 0x4E00 && cp <= 0x9FFF) return true;
        // CJK 扩展 A 区 (U+3400 ~ U+4DBF)
        if (cp >= 0x3400 && cp <= 0x4DBF) return true;
    }
    return false;
}
```

## 三、缓存系统（第三期）

两级缓存架构：L1 进程内分片 LRU + L2 Redis。

```
请求 → L1 (ShardedLRU, ~ns) → hit? → 返回
                                miss? ↓
                              L2 (Redis, ~ms) → hit? → 回写 L1 → 返回
                                                  miss? ↓
                              计算 (KeywordService/WebSearchService)
                                    → 写入 L1 + L2 → 返回
```

### 3.1 分片 LRU (L1)

```cpp
// include/common/LRUCache.h — ShardedLRUCache<K,V>
// hash(key) % N → N 个独立 LRU 实例, 每片一把 shared_mutex
template<typename K, typename V>
class ShardedLRUCache
{
    bool get(const K& key, V& value) {
        size_t idx = std::hash<K>{}(key) % _shards.size();
        auto& s = *_shards[idx];
        std::shared_lock lock(s.mutex);      // 读共享锁
        return s.cache.get(key, value);
    }

    void put(const K& key, const V& value) {
        size_t idx = std::hash<K>{}(key) % _shards.size();
        auto& s = *_shards[idx];
        std::unique_lock lock(s.mutex);      // 写独占锁
        s.cache.put(key, value);
    }

private:
    struct Shard {
        LRUCache<K, V> cache;                // list+map 纯数据结构
        mutable std::shared_mutex mutex;     // 每片独立锁
    };
    std::vector<std::unique_ptr<Shard>> _shards;  // unique_ptr 避免移动
};
```

> **为什么用 `unique_ptr<Shard>` 而非 `vector<Shard>`?**
> `Shard` 含 `shared_mutex` (不可移动/复制)，`vector` 扩容时需要移动元素。
> `unique_ptr` 可移动，`vector<unique_ptr>` 扩容只移动指针。

### 3.2 Redis 缓存 (L2)

```cpp
// src/common/CacheManager.cpp — 两级缓存调用链
bool CacheManager::getKeyword(const std::string& query, std::string& result)
{
    if (_kwCache.get(query, result))        // ① L1 命中 → 返回
        return true;

    std::string l2key = "ns:kw:" + query;   // ② L2 查找
    if (redisGet(l2key, result)) {
        _kwCache.put(query, result);         // ③ 回写 L1
        return true;
    }
    return false;                            // ④ 两级都 miss
}

void CacheManager::putKeyword(const std::string& query,
                               const std::string& result)
{
    _kwCache.put(query, result);             // 写 L1
    redisSet("ns:kw:" + query, result, 3600);// 写 L2 (TTL=1h)
}
```

> **Redis 不可用时**: `initRedis()` 连接失败 → `_redis` 为 `nullptr` →
> `redisGet/Set` 直接返回 `false` → 自动降级为纯 L1。服务不中断。
